# AI Usage Documentation

## Tools Used
- Claude for Flask routes, HTML/CSS, debugging, and code review

## Prompts
- "Help me generate a Flask route for my project that follows the CRUD pattern from my SQL file."
- "I wanted to include an error handling that ensures the link a user inputs is a valid LinkedIn URL. How should I approach?"
- "Can you read my html files and do the following: 1) Identify any errors in the code, 2) Suggest improvements to both the code and the design of my pages?"

## What Worked
- The flask routes generated by Claude were mostly correct, and only required some minor adjustments to work with my specific database schema.
- Generated front-end code that was mostly correct, but required some adjustments to fix many of the errors and improve the design of the pages.


## What I Changed 
- The agent did break a few functions in my code, but I was able to fix some of the issues. 

## Lessons Learned
- I learned that AI agents often hallucinate and, if not used carefully, can lead to more work than they save. 
- I also learned to always review the code generated by the AI agent to esure it is correct and does not introduce any bugs. 
- AI is great when creating a draft of a function, but should not be solely relied on to create a final product.
